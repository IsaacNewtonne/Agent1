# User Personas and Jobs To Be Done

## Persona 1: Local AI Developer

### Description

A developer running local models through Ollama who wants to build practical agents.

### Jobs

- Create agents quickly.
- Test prompts and tools.
- Run coding workflows locally.
- Inspect exactly what the agent did.
- Avoid API costs.

### Pain Points

- Existing frameworks are Python-heavy.
- Tool calls are hard to audit.
- Local model support is inconsistent.
- Multi-agent systems are hard to debug.

## Persona 2: Open-Source Builder

### Description

A builder creating reusable tools or agents for the community.

### Jobs

- Package example agents.
- Build tools and integrations.
- Share configs.
- Contribute to a Rust runtime.

### Pain Points

- Existing agent ecosystems are fragmented.
- Hosted APIs make examples hard to reproduce.
- Plugin systems are inconsistent.

## Persona 3: Privacy-Conscious Power User

### Description

A user who wants private AI automation on their own machine.

### Jobs

- Run local assistants.
- Use local files safely.
- Keep data off the cloud.
- Control memory and logs.

### Pain Points

- Hosted AI tools require uploading data.
- Local tools are often too technical.
- Permissions are not clear.

## Persona 4: Small Team Operator

### Description

A small team that wants a private agent platform on a local server or VPS.

### Jobs

- Run internal agents.
- Review agent actions.
- Store sessions.
- Connect tools through MCP.
- Avoid vendor lock-in.

### Pain Points

- Enterprise platforms are expensive.
- Cloud compliance is difficult.
- Multi-agent workflows are hard to govern.

## Persona 5: Researcher

### Description

A researcher exploring multi-agent coordination and protocol design.

### Jobs

- Create agent teams.
- Test planner-worker-critic flows.
- Inspect message traces.
- Compare model backends.

### Pain Points

- Most systems hide internal state.
- Reproducibility is poor.
- Tool and memory behavior is inconsistent.
